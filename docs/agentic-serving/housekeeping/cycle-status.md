## Settled Premises (carried into Cycle 8)

1. **Target form (practitioner, firm yes, 2026-07-01):** the agentic serving strategy is a declarative ensemble on the L0 engine, not bespoke imperative Python.
2. **Governing invariant (candidate ADR-044 — the first DECIDE item):** agentic serving composes as llm-orc-native ensembles (model-profiles + scripts + engine primitives). Control flow lives in the ensemble, not in an imperative driver. Where the engine is inadequate, **extend the engine** (new primitives), never build a parallel orchestration layer beside it.
3. **Removal strategy — clean-slate (practitioner, 2026-07-01):** the ~12.7K-LOC bespoke `agentic/` layer and the retained spikes are learning/reference, not a substrate to harden. Cycle 8 builds the declarative serving product fresh (informed by the spikes) and deletes the bespoke layer as the new form reaches parity. No strangler pattern on spike code. Rationale: "We've learned a lot, but I don't want to do a strangler pattern on spike code." (Warrants ADR-045-shaped record with rejected alternatives: strangler, parity-then-delete.)
4. **Ensemble-over-frontier lens (practitioner, 2026-07-02):** whenever the reasoning reaches "a small model can't do this," first ask what **ensemble orchestration** (bounded roles + structure + deterministic verification) can achieve before defaulting to "a frontier model should do it." Composition is the capability lever (north star); a frontier model is a fallback and a benchmark, not the reflex. Wired into ADR-047 (the composer's compose step) and applies corpus-wide as a design lens.

## Feed-Forward Signals

### From Cycle 8 DECIDE (ADR-046 accepted 2026-07-02) → Q4 extensibility + Q2 grounded acceptance

**Settled premises Q4 builds on (do not reopen):**
1. **The dispatch mechanism is shipped, not Q4's to build.** Dynamic dispatch is an L0 engine primitive (runtime-resolved `ensemble:` target, resolved at the phase layer via the shared `${dep.field}` resolver). Q4 builds the *registry the target resolves against* + the binding, not the resolution mechanism.
2. **The registry's baseline is operator-curated** (operators register candidate seats per capability). What ADR-046 retired is one specific form — a *persistent orchestrator LLM* authoring ensembles at runtime via a `compose_ensemble` tool (AS-6 / ADR-006). **Runtime ensemble authorship is NOT ruled out** (practitioner, 2026-07-02): composing ensembles from other ensembles at runtime is a live direction of interest, shape deferred. The open boundary is *form* — it must be a **declarative engine capability** (AS-11: extend the engine — e.g. a runtime-composition primitive building on dynamic dispatch), not a resurrected persistent-LLM-actor-with-a-tool-surface. The line to keep clean: what *decides* the composition at runtime — a declarative guard/dispatch/decider-seat, vs. an unbounded LLM authoring step.
3. **AS-2 (validate-before-load) is the registry's admission gate.** Every candidate is validated against the ensemble reference graph (no cycle / within depth / resolves to an existing entry) before it can fill a seat. Surviving invariant to *wire*, not re-derive.
4. **Trust-promotion stays retired — and it is separate from runtime authorship.** AS-5's cross-session stabilization/promotion half was retired with the calibration subsystem (no successor). Composing ensembles at runtime (premise 2) is a live direction; *accumulating quality signals to auto-promote candidates across sessions* is the retired calibration machinery. Keep them distinct — the former is welcome, the latter is what ADR-046 dissolved. (Per-dispatch quality signal is Q2's concern.)
5. **Keyed on the Topaz capability taxonomy** (grounded 8-skill set: `code_generation`, `tool_use`, `mathematical_reasoning`, `logical_reasoning`, `factual_knowledge`, `writing_quality`, `instruction_following`, `summarization`). classify emits `{target}` → dispatch resolves it against the Topaz-keyed registry.
6. **Extend the engine, never a parallel layer (AS-11).** The registry is engine/config, not a bespoke orchestration surface.

**The open fork Q4 owns — binding time + composition depth:** (a) *binding time* — load-time curation vs. turn-time selection vs. hybrid; and (b) *composition depth* — selecting an existing registered ensemble vs. **runtime composition of a new ensemble from parts** (the practitioner's runtime-authorship interest, 2026-07-02; shape deferred, form must be declarative per premise 2). Plus how operators register/extend, and the serving default (the default generalist ensemble + the operator-extension path). ADR-046 §Open Q4.
