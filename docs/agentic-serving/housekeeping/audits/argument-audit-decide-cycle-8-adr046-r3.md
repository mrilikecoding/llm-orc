### P1 — Must Fix

**P1-new-1. Row 014's new standalone rationale — "replaced by the seat contract's per-dispatch pass/fail" — asserts an unexamined equivalence between two structurally different signal types, and doesn't take up R2's own offered alternative (cross-reference it from §Open's Q2).**

- **Location:** ADR-046 §3 row `014` (line 58): "**superseded** — the in-process per-dispatch calibration verdict (Proceed/Reflect/Abstain, AUQ/HTC trajectory features) is replaced by the seat contract's per-dispatch pass/fail (§2). This is 014's own rationale — the per-dispatch mechanism — not folded under 007's cross-session trust-transition (R2 P1: unbundle 014 from 007)." Compare §2 (line 33): "The seat contract is the existing `core/validation/` framework (`ValidationConfig`: structural/schema/behavioral/quantitative/semantic layers; pass/fail `ValidationResult`)... **deterministic-first** — the semantic (LLM-judge) layer is optional and soft; seat eligibility rests on the deterministic layers." Compare ADR-014 §Decision directly: the quality signal is a composition of post-hoc result-check + "**in-process AUQ confidence** — verbalized confidence emitted by the dispatched ensemble's component agents, propagated through attention as a soft constraint (System 1) and gated at a binary threshold... (System 2)" + "**HTC trajectory features** — process-level features extracted across the entire dispatch trajectory: token-level entropy patterns, attention-weight distributions over tool-call sequences, decision-confidence trajectories across reasoning steps," with an Abstain trigger on "entropy collapse: token-level entropy in the dispatched ensemble's most recent N tokens drops more than 1.5 standard deviations below the trajectory's running mean," applying "to *every* dispatch decision... for all ensembles regardless of calibration status."
- **Claim:** `core/validation/`'s deterministic-first, output-contract pass/fail check is a replacement for AUQ/HTC's process-level, generation-time confidence signal.
- **Evidence gap:** These are different signal types answering different questions. `core/validation/` validates the *output* against a declared contract (did the result conform to schema, pass structural/behavioral/quantitative checks). ADR-014's mechanism is a *process*-level signal derived from the model's own generation trajectory (token-entropy, attention-weight distribution, verbalized confidence) — designed specifically to catch failure modes an output-correctness check cannot see: a syntactically valid, contract-conforming output produced via a low-confidence, entropy-collapsing, or otherwise anomalous generation trajectory. Nothing in §2's seat-contract description operates on trajectory data at all; the semantic (LLM-judge) layer named as the closest analog is explicitly "optional and soft," and seat eligibility "rests on the deterministic layers" — i.e., exactly the output-only layers that cannot see what AUQ/HTC are designed to catch. The row's claim that one *replaces* the other is asserted, not argued. This is the same pattern this audit sequence has now caught three times: R1's P1-3 (AS-5's cross-session stabilization mechanism dropped under "the principle stays"), R2's P1-2 (AS-2 dropped under ADR-006's compose_ensemble supersession) — both cases where a genuinely actor-agnostic mechanism was retired by proximity to an actor-bound sibling, not by an argument that it was actually superseded on its own terms. ADR-014's mechanism is no more actor-bound than AS-2 was: nothing about token-entropy monitoring or trajectory-confidence gating presupposes a persistent orchestrator actor; it presupposes only that *some* dispatch is happening, which is exactly what still happens under the seat mechanism. R2's own recommendation offered a second option — cross-reference ADR-014 from §Open's Q2 ("Grounded acceptance beyond contract assertions... whether more is needed is a PLAY/BUILD validation target") as a candidate mechanism for "the more." That cross-reference was not made; Q2's text still does not name ADR-014, AUQ, or HTC anywhere in the document (confirmed by direct search).
- **Recommendation:** Either (a) argue explicitly why AUQ/HTC's trajectory-level signal is unnecessary once deterministic contract checks are in place — e.g., if the practitioner's view is that seat-contract failures are sufficient and process-level anomaly detection is not worth the mechanism's cost, say so and accept the tradeoff explicitly — or (b) add ADR-014 to §Open's Q2 as a named candidate mechanism for the "beyond contract assertions" signal the accept/another-round decision may still need, consistent with how AS-2 and AS-5 are now handled (named, not silently dropped).

### Question 1: What alternative framings did the evidence support?

**The per-component-vs-per-ADR inversion R2 surfaced is now resolved for most, but not all, of the rows it was raised against.** R2's central framing finding was that "is this ADR's actor-dependent mechanism superseded" and "does everything this ADR decided depend on the actor" are different questions, and the P2-6(R1) fix had answered only the first while presenting itself as having answered the second. This round's component-by-component pass confirms the inversion is now correctly resolved for rows 001, 006, and 008 (the AS-2 split is exactly the component-level answer the inversion calls for; 008's flat supersession is now independently re-derived, not just inherited, as genuinely correct). But row 014 is a fresh instance of the same failure the inversion names: the row was restructured (unbundled from 007, given its own rationale) in direct response to R2's finding, yet the new rationale itself commits the identical error — asserting that an actor-agnostic, process-level mechanism (AUQ/HTC trajectory monitoring, which no more depends on "the orchestrator" as a persistent actor than AS-2's reference-graph validation did) is subsumed by a different, narrower mechanism (deterministic output-contract checking) without examining whether the narrower mechanism actually covers what the broader one caught. The pattern is not fully internalized as a *checking discipline* yet — it has been applied successfully to some rows and, in the same revision, failed to catch its own recurrence in an adjacent row it directly touched.

**A second, narrower alternative framing:** the row-level fixes in this revision read as responsive to R2's *specific named findings* (AS-2, ADR-001, ADR-002, the 003/009-010 contradiction) more than as a *general re-audit of the reclassified block* using the pattern those findings established. This is visible in the asymmetry between how thoroughly 006 was reworked (a full split with explicit Q4 tie-in) versus how row 014 was reworked (restructured but not re-examined against the same standard) and row 003 (touched extensively for the query_knowledge/record_outcome fix but not swept for the other three tools in the same table).

**Dominant framing:** the component-by-component discipline P2-6(R1) → R2's four P1s → this round's fixes has now converged; the remaining `[confirm]`-turned-`superseded` rows are settled.
