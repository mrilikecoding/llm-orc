# Layer-Conditional Composition for Cheap-Orchestrator + Ensemble Architectures

*2026-05-04*

## Abstract

This essay reports findings from Cycle 4 of the agentic-serving scoped corpus. The cycle's question was what design methods the cheap-orchestrator + ensemble pattern needs in order to extend from manually-staged multi-stage workflows (Cycle 3 Spike D) toward an agentic system capable of driving arbitrary coding sessions over long horizons — with "running a full RDD cycle using the agentic-serving flow itself" as the concrete capability benchmark — and which of those design methods are properties of ensembles specifically versus orchestration more generally. The research process ran three literature reviews — on long-horizon agent reliability, composition shapes per layer, and long-horizon reliability infrastructure (approximately 66 sources combined across the three waves: 23 + 20 + 23 per the source tables) — an analytical decomposition mapping RDD-cycle phase demands to architecture layers, and a behavioral spike validating the architecture's `invoke_ensemble` dispatch path via `llm-orc serve` direct API. The cycle's central finding is that the pattern's value is not located in any single layer; it is **layer-conditional cross-layer composition where components have measurably-different error distributions**, with **long-horizon reliability infrastructure operationalizable within the existing four-layer architecture** rather than requiring architectural reorganization. Seven concrete design decisions surfaced as candidate ADRs, including one amendment to the architecture's layering rule (an upward read-only signal channel from L0 to L1 enabling cross-layer calibration). The local-first commitment that distinguishes the cycle's deployment from the published literature's predominantly-frontier-tier evidence is operationalized as decompose-complex-decisions-into-many-bounded-stateless-per-member-tasks, with cheap-cloud-orchestrator handling routing and summarization and local models amplifying deterministic and bounded-scope work.

## The Starting State

Cycle 4 of the agentic-serving scoped corpus opened with three prior cycles' empirical anchors and a domain-model commitment to the four-layer architecture (orchestrator + script-models + ensembles + plexus, per ADR-002). Cycle 1 validated the hybrid CAP-9 deployment shape (cloud orchestrator + local ensembles) as the configuration that closes the per-turn-context throughput bottleneck local-only deployments hit on consumer hardware. Cycle 2 produced essay 003's four-axis design-priorities frame and three spikes — including Spike A3, a novel ensemble combining a script-based deterministic analyzer with two heterogeneous cloud LLM reviewers and Python-harness-enforced MARG-style concatenation, demonstrating that some well-architected ensembles produce capabilities prompt-steering structurally cannot replicate. Cycle 3 ran a three-arm architecture comparison on cross-file verification (Spike C) and a multi-stage workflow pilot (Spike D). Spike C produced the cycle's strongest central-question evidence: the architecture's `+ orchestration` primitive (script-agent + LLM reviewer ensemble dispatched via `invoke_ensemble`) caught the actual cross-file value mismatch in 3 of 3 trials, where cheap-bare with general agentic tools caught it in 1 of 3 (variable on autonomous tool use) and frontier-bare in 0 of 2 (single-shot has no access to other files). Spike D's pilot extended the finding to multi-stage workflow at zero-dollar cost matching frontier-bare quality, with the substantive deployment-shape finding that the OpenCode CLI shell-subprocess workaround stalls indefinitely on substantial code-containing prompts while the production `ConfigurationManager` + `ModelFactory` path completes the same workflow in 24.8 seconds (the opencode-CLI stalls were sustained: four attempts ranged from 14 minutes to 14 hours before manual termination). Cycle 3 closed in Mode B (Research Only) with five research logs as the deliverable; the closing susceptibility snapshot recommended three grounding actions for Cycle 4 entry: distinguish "deterministic tool output" from "ensemble orchestration" on cross-file verification, name the autonomous-routing evidence gap explicitly, and test frontier with matched information access as a comparison-baseline scope condition.

The corpus also carries the local-first commitment as a load-bearing value: the deployment is deliberately positioned for local-model leverage under cheap-cloud-orchestrator routing, not for cloud-only frontier-tier capability. Essay 002's bilateral observability finding from Cycle 1's PLAY phase (user passivity in face of silent failure as the dominant experiential mode) and essay 003's "default-not-ceiling" reading of ADR-011 (the orchestrator-LLM-is-a-Model-Profile decision is defensible as default but not as ceiling) further constrained Cycle 4's design space. The architectural artifacts at cycle entry included eleven accepted ADRs, a system design document (v2.0 with companion `system-design.agents.md` per the v0.8.5 companion-file pattern), product-discovery, domain-model with fourteen invariants, scenarios, interaction-specs, and a roadmap with WP-K (Plexus-active paths) and WP-J (Bootstrapping pipeline) deferred.

## Research-Entry Framing and Constraint-Removal

Cycle 4's research-entry moment applied the question-isolation entry protocol (ADR-082 of the RDD plugin methodology — distinct from the agentic-serving corpus's ADRs 001–011 — establishes the five-step entry protocol where the practitioner articulates research questions before the agent reads the corpus, the agent composes a constraint-removal prompt against the most consequential existing artifact, and a research-methods-reviewer audits the question set before any research dispatch) before reading the existing artifact corpus. The practitioner articulated the cycle's primary research question and six sub-questions decomposed from it; the agent named the four-layer architecture commitment in `system-design.md` as the most consequential existing artifact bracketing for the constraint-removal prompt. The practitioner's response engaged both the broader bracket (architecture-removed) and the tighter bracket (`invoke_ensemble`-only) and produced three commitments that shaped the cycle's research surface.

The first commitment is the operationalization of the local-first value as load-bearing: any design method that recovers capability by pushing more work to the cloud orchestrator — even if functionally adequate — fails the cycle's value test. The bare-orchestrator collapse case is rejected on value grounds, not on technical grounds. The practitioner stated this directly: cloud-orchestrator-only mode is "not much different from using something like Sonnet, though it's cheaper" — failing the local-leverage value the architecture exists to deliver.

The second commitment recognizes that `invoke_ensemble`-alone is partial. The tighter bracket gets the architecture part of the way, but its viability depends on (a) shipping ensembles that compose what is needed, and (b) the orchestrator reliably selecting the right ensemble. The first is a coverage problem; the second is a reliability problem — and the practitioner transferred a finding from the RDD methodology corpus itself: structural hooks at phase transitions were ultimately necessary in RDD because as context grows, an agent's judgment to follow a directive degrades. The transfer's plausibility is shared mechanism (context growth degrades salience of prior directive), but the decision-class differs: phase-transition decisions are one-shot procedural at known points; ensemble selection is continuous routing under variable inputs. Whether the failure mode transfers is itself an empirical question, recorded as the cycle's added Sub-Q6.

The third commitment is the North-Star benchmark: drive a full RDD cycle using llm-orc's agentic-serving flow. The benchmark is demanding by design — long-horizon, multi-modal (research, modeling, decision, build, debug, reflect), self-eating dogfood. Whatever design methods the cycle proposes must scale to that shape, not only to the bounded code-review surface where the pattern has been demonstrated.

The research-methods reviewer's audit of the question set surfaced seven flags, all accepted with reformulations. Most consequentially, the reviewer identified that the original Sub-Q6 phrasing ("what harnesses can enforce ensemble selection") embedded the conclusion that harnesses are the right intervention class, and that the literature on agent reliability distinguishes intervention classes that work at finitely-occurring decision points (where Spike A3's MARG-aggregation pattern is class (a) deterministic-override) from those that work for continuous routing (where the appropriate class is decomposition or governed retrieval). Sub-Q6 was reformulated to place the transfer-test ("does the failure mode actually occur in this architecture at these session lengths?") before the intervention-class question.

## Long-Horizon Reliability — What the Literature Settled

The first lit-review covered long-horizon agent reliability, judgment decay under context growth, intervention-class taxonomy, and long-horizon coding-agent design. The macro empirical picture is settled. Khanal, Tao, and Zhou (arXiv:2603.29231, 2026) establish across 23,392 episodes on 10 open-source models that pass@1 drops from 76.3 percent on short tasks to 52.1 percent on very-long tasks, and that the decay exceeds geometric prediction — meaning errors are positively correlated across steps, not independent. Software engineering tasks have the steepest degradation curve of any domain tested (GDS — Graceful Degradation Score — 0.90 → 0.44), directly covering the North-Star benchmark's task class. Khanal et al.'s most counterintuitive finding for the cycle's framing is what they term the meltdown-on-paradox: frontier models exhibit the *highest* meltdown rates because they pursue ambitious strategies that spiral when exploratory paths fail. DeepSeek V3's 19 percent meltdown rate at very-long horizons is higher than smaller open-weight models that fail earlier and less catastrophically. The "frontier" in Khanal et al. is the frontier of an open-source models cohort (DeepSeek V3, MiniMax M2.5, and others), not proprietary frontier models (GPT-5, Claude Opus 4.5); extrapolation from open-source-frontier meltdown rates to proprietary frontier behavior on the North-Star benchmark is directionally plausible but not directly evidenced. The MOP paradox complicates simple capability-tier reasoning for local-first deployments — at the open-source cohort level, cheap-orchestrator + good-architecture may be more reliable than frontier-of-cohort on long-horizon work, and "throw a more-capable model at the hard cases" is not the reliable backup the practitioner intuition might suggest. An alternative reading available in the same data is worth recording: the cheap orchestrator's reliability advantage on long-horizon may be partly due to capability-bounded failure (failing before the meltdown threshold) rather than architectural reliability compensation. Khanal et al. characterize weaker models as failing "earlier and more uniformly" — distinct from failing *less*. The architectural-reliability and capability-bounded-failure readings coexist in the data; neither is settled.

The context-rot finding from Chroma's 2025 measurement extends Liu et al.'s "Lost in the Middle" (TACL 2023) to a practitioner-level result: significant degradation occurs at 50,000 tokens inside a 200,000-token window — well before overflow. The mechanism is attention dilution following the U-shaped positional bias, compounded in agent sessions by accumulated tool outputs and exploration artifacts. This is consequential for the cycle: the orchestrator's per-turn schema overhead from the closed five-tool surface plus client-declared tools is approximately 15,000 tokens; by mid-session, earlier tool results sit in the mid-context attention valley.

The cycle's added Sub-Q6 transfer-test question received a partial answer from the literature. CAAF (Zhang, arXiv:2604.17025) proposes structurally different mechanisms for the two cases: State Locking for settled iterative decisions (one-shot procedural, like RDD's phase-transition hooks), and context firewalls for continuous routing isolation. Phase-transition decisions and continuous-routing decisions share a mechanism — context growth degrades salience of a prior directive — but differ in decision frequency. **No paper directly studies ensemble routing judgment degradation at high decision frequency as a distinct experimental phenomenon.** This is the cycle's novel empirical territory; the literature does not settle whether continuous routing degrades the same way as phase-transition decisions.

The intervention-class taxonomy that emerged from the lit-review distinguishes four classes with uneven evidence. Class (a) — harness-level deterministic override — is the best-evidenced intervention class. CAAF achieves 100 percent reliability at $0.0027 per artifact versus 0 percent for monolithic GPT-4o on paradox-detection tasks. The key finding from CAAF's authors is that "apparent LLM reliability in safety-critical domains is often a prompt engineering artifact" — structural enforcement is qualitatively different from prompt guidance. Class (a) works best at finitely-occurring decision points where topology is pre-specifiable. Tool schema gating (where write tools are simply absent from the planner's interface, as in OpenDev's plan-mode) is a class (a)/(c) hybrid that avoids constrained decoding's alignment tax: Zhou (arXiv:2604.06066) reports Qwen3-8B drops from 50 percent to 38 percent accuracy under grammar constraints, with structure consuming cognitive budget needed for semantic quality.

Class (b) — context-management primitives — requires governance rather than passive storage. The naive memory-scaffold pattern is contraindicated by Khanal et al.'s finding of universal non-improvement (six models showed negative effects, four were neutral, none improved) from episodic memory augmentation. HELM (OpenReview, January 2026) represents the most advanced class (b) approach the cycle's lit-review accessed: three-tier SHNM memory with epistemic governance and provenance-tracked retrieval with recency and status re-ranking. The principle from the externalization taxonomy (Zhou et al., arXiv:2604.08224) is that context-layer externalization fails across sessions and with lost-in-the-middle degradation as context lengths saturate; harness-layer externalization is necessary for long-horizon tasks.

Class (c) — architecture decomposition — is the most reliably effective but requires pre-specifiable topology. The best-practice coding-agent architecture has converged across academic (OpenDev, Bui 2026) and practitioner (Anthropic engineering) sources on a single pattern: convert long-context problems into retrieve-on-resume problems via external structured state — progress files, feature JSON, git history. Each session is bounded with explicit structured handoff rather than accumulated context. This is the most consistently supported mechanism in the reviewed literature for long-session coding-agent continuity.

The Spike A3 generalization question received a direct answer at this stage. Spike A3's MARG-aggregation harness is class (a) at a *finitely-occurring* aggregation decision; continuous ensemble routing is a high-frequency routing decision. **The intervention-class analysis says these need different approaches — class (c) decomposition (pre-compiled routing policy, schema-level enforcement) or class (b) governed retrieval, not a repeated class (a) wrapper at every step.** The cycle's reformulated Sub-Q6 inherits this finding directly: the right intervention class for ensemble routing under context growth is decomposition or grounding, not per-decision wrapping.

The benchmark picture for long-horizon coding agents is stark. SWE-EVO (Thai et al., arXiv:2512.18470) documents a 47.8-point drop from short-horizon (72.8 percent SWE-Bench Verified) to long-horizon multi-file evolution tasks (25 percent on SWE-EVO). Even top proprietary models score only 23 percent on SWE-Bench Pro's enterprise-difficulty tasks. Failure modes are qualitatively distinct from short-horizon failures: shallow exploration, no backtracking (absent from OpenHands baseline), loop entrapment.

The local-model capability ceiling found by the lit-review is consequential for the cycle's local-first commitment. Open-weight frontier models have converged with frontier proprietary models on short-horizon isolated coding tasks: Qwen3-Coder-Next 70.6–71.3 percent on SWE-Bench Verified versus frontier proprietary at approximately 74–77 percent. The gap persists on long-horizon tasks: 56.2 percent versus 62.5 percent on SWE-Bench Pro. Terminal-Bench 2.0 shows a 24-point gap between Qwen3-Coder-Next (34.2 percent) and Claude Opus 4.5 (58.4 percent) on agentic terminal tasks. The 7B–14B reliability boundary is the load-bearing finding for *locally-deployed* small models: these models reliably handle single-file, bounded, stateless tasks but unreliably handle multi-step tool calling after context accumulation. The failure mechanism is structured-output format maintenance competing with semantic reasoning capacity under context pressure. The 32B boundary is where local models become competitive with frontier on many coding tasks. The Qwen3-Coder-Next 70.6–71.3 percent SWE-Bench Verified score cited above is from cloud inference of a model substantially larger than the 7B–14B reliability class; that score does not establish that the 7B–14B local reliability boundary has shifted, only that the open-weight family continues to scale at larger sizes on cloud inference. The local-deployment 7B–14B class remains the binding empirical reference for the cycle's local-first commitment. The lit-review's operationalization of the local-first commitment is precise: local models amplify deterministic and bounded-scope ensemble tasks (the territory where Cycle 3 grounding action 1 already located the mechanism); the cheap cloud orchestrator handles routing, summarization, and decisions requiring sustained multi-step reasoning. Ensemble designs that decompose complex decisions into many bounded stateless per-member tasks put local-first on the firmest available empirical ground.

## RDD-Cycle Decomposition and Mechanism Attribution

The cycle's analytical-decomposition pass walked the RDD methodology phase by phase to identify cognitive surface, long-horizon session demand, decision-class mix, and hardness for the cheap-orchestrator + ensemble pattern. The decomposition produced a load-bearing structural finding: RDD phases break cleanly into three clusters by dominant decision-class, and the clustering is consequential for how the cycle's Sub-Q6 transfer test resolves at each phase.

Cluster 1 phases — RESEARCH, DECIDE, SYNTHESIZE, and the invariant-amendment portions of MODEL — exhibit heavy specialist-dispatch at known transition points. The orchestrator's primary job is correct dispatch at structurally-determined moments. Audit dispatches, lit-reviewer invocations, conformance scans, susceptibility snapshots — all fire at specific phase positions in the skill text. The Sub-Q6 transfer of RDD's structural-hook finding is most plausible here: the decision-class match is direct (one-shot procedural at known points), and the failure mode RDD documented (context growth degrades agent judgment to follow a directive) is the exact failure mode this cluster faces.

Cluster 2 phases — BUILD and ARCHITECT, plus portions of DEBUG and REFACTOR — exhibit heavy continuous routing under variable inputs. The orchestrator's primary job is moment-to-moment routing under inputs that are not predictable from the phase position: which file to read, which test to run, when to commit, when to mode-shift between debug and refactor and review. The Sub-Q6 transfer is least plausible here. The decision-class differs (continuous routing, not one-shot procedural), and the intervention class likely shifts toward context-management primitives (working-memory consolidation, sliding-window summarization, retrieval-grounded routing) rather than harnesses at specific decision points.

Cluster 3 phases — DISCOVER, PLAY, and conversational portions of MODEL — are conversational and exploratory. The orchestrator's primary job is responsive depth in conversation with the practitioner. Ensemble role is limited; the architecture's value comes from orchestrator plus read-tools plus memory.

The North-Star benchmark demands all three clusters in sequence. This is consequential: design methods that work for one cluster may not transfer to another. The methodology-design surface is therefore plural, not singular — which is itself a refinement to the cycle's research-entry framing.

The cycle's mechanism-attribution pass mapped the empirical evidence base — concentrated in Cycle 3's Spike C and Spike D plus Cycle 2's Spike A and Spike A3 — to the four-layer architecture for each demand class. For cross-file verification, the load-bearing layer is script-models (deterministic file access); the ensemble role is wrapper, not mechanism (Cycle 3 grounding action 1 confirms). For documentation review with per-recommendation specificity, the load-bearing mechanism is the composition pattern itself — MARG-style concatenation preserves what cascades destroy, and heterogeneity-uncorrelated-errors layers on top (Spike A3 evidence; multiple trials). For multi-stage workflow with deterministic verification, the cross-layer composition is load-bearing (orchestrator routes between specialized ensembles; script-models verify; LLMs synthesize) — though Spike D was manually staged and autonomous routing remains an evidence gap. For long-horizon multi-turn coding, no layer of the four-layer architecture directly addresses long-horizon reliability primitives (working memory, retrieval-grounded routing, sliding-window summarization); this is the cycle's largest framing-shift candidate identified at the analytical-decomposition stage. For routing intelligence, the cheap-orchestrator's tool-choice variability (Spike A's 1-of-3 trials with autonomous Read use) suggests routing intelligence is real but variable at the cheap-tier — and Cycle 3 grounding action 2's evidence gap remains until directly tested.

The unifying mechanism the corpus has surfaced (essay 003, "What this means for the cycle") is **composition of components whose error distributions are different enough that the composition's coverage exceeds any single component's**. Spike A3 instantiates this through heterogeneity-across-LLM-families. Spike C instantiates it through deterministic-versus-probabilistic complementarity within an ensemble (script has zero error distribution where LLM has nonzero). Spike B refined it as task-class-dependent. The unifying frame is consequential for Sub-Q3's primary scoping decision: the architecture's value is not located in a single layer; the load-bearing layer per task class is determined by which complementarity surface the task most exposes.

The analytical decomposition produced a tentative Sub-Q3 candidate finding: a refinement of the research-entry hypothesis-D (cross-layer composition is the irreducible unit). The refined version is that the irreducible unit is composition-of-components-whose-error-distributions-differ, but the components themselves are layer-typed. This makes Sub-Q4's design-method scope plural: composition shapes for ensembles, composition shapes for script-models, composition shapes for orchestrator routing — and the cross-layer composition shapes that bind them.

## Composition Shapes Per Layer

The second lit-review covered composition shapes per layer for the cheap-orchestrator + ensemble pattern, with explicit attention to recursive ensembles-of-ensembles (the well-scoped Cycle 4 seed essay 003 named explicitly), typed contracts as a composition-validator pattern, generative topology construction, calibration-gated routing, and retrieval-grounded selection. The review built on Cycle 2 Loop 4's coverage of MARG-style concatenation, heterogeneity-uncorrelated-errors, and Yao 2025's panel-discussion diversity-destruction without re-covering that ground.

The strongest finding for the cycle's ensemble-design surface is the strengthening of the no-collapse-summarization principle. DeliberationBench (arXiv:2601.08835, December 2025) finds 82.5 percent win rate for best-single selection versus 13.8 percent for the best deliberation protocol — a 6.0× gap at 1.5–2.5× higher compute. This substantially strengthens Cycle 2 Loop 4's Yao 2025 panel-discussion finding from "documented anti-pattern" to "documented reliability risk." The practical implication is direct: parallel independent roles with concatenation are the only defensible ensemble topology. Any sequential deliberation step is a documented reliability risk. Spike A3's MARG-concatenation pattern, harness-enforced rather than orchestrator-compliance-dependent, is now strongly literature-supported.

The Self-MoA finding (arXiv:2502.00674, ICLR 2025) challenges the cycle's heterogeneity prior — sampling multiple times from a single best-performing model outperforms cross-family mixing by 6.6 percentage points on instruction-following benchmarks — but the challenge is task-class-conditional. Self-MoA's finding holds where the task requires correctness on well-defined questions; it does not hold where the task requires cross-family coverage of systematically different blind spots, which is the territory of Cycle 2's Spike A3 documentation review and Cycle 3's Spike C cross-file verification. The cycle's task class on the empirically-grounded surfaces (verification, documentation review, code review) is the cross-family-coverage class. The resolution is *cluster-conditional within the RDD benchmark*, not universal across it: Cluster 1 phases (RESEARCH, DECIDE, SYNTHESIZE) and the verification work in Cluster 2 phases (BUILD's review-and-debug; ARCHITECT's conformance check) sit in the cross-family-coverage class; Cluster 3 phases (DISCOVER, PLAY, conversational portions) and the instruction-following moments inside any cluster may sit in Self-MoA's territory. The composite framing should be read as cluster-conditional on this dimension.

For recursive ensembles-of-ensembles, the cycle's question is whether the literature offers a deployable pattern. RecursiveMAS (arXiv:2604.25917, April 2026) achieves +8.3 percent accuracy and 34.6–75.6 percent token reduction across benchmarks via latent-space recursive computation, but requires gradient access for co-optimization — the mechanism is unavailable in llm-orc's black-box inference setting. Attention-MoA (arXiv:2601.16596, January 2026) is the deployable alternative: inter-agent critique-and-refine within each layer plus residual history accumulation prevents the collapse-at-depth problem standard MoA exhibits after layer 3. Attention-MoA achieves 91.15 percent versus MoA's 88.56 percent on AlpacaEval 2.0 with monotonically increasing performance to layer 5, no fine-tuning required.

The Attention-MoA finding carries a binding scope condition for the cycle's "cheap-orchestrator + ensemble" framing: on AlpacaEval 2.0 and MT-Bench (instruction-following benchmarks, not code-review or verification), the aggregation-agent quality drives a 12.82-percentage-point gap in outcomes; the cloud orchestrator quality is the bottleneck, not the member models. The dependency may differ on the cycle's primary task class (code-review and verification) where ensemble member outputs include deterministic-tool findings rather than only LLM-generated content. For the cheap-orchestrator + ensemble pattern, this means **the orchestrator's role at the aggregation moment is plausibly load-bearing for ensembles-of-ensembles success — at least on instruction-following territory; code-review/verification may show different dependency profile.** "Cheap" must be calibrated against this — if the cheap orchestrator is the aggregator, its quality ceiling may cap recursive ensemble depth on instruction-following tasks; whether the same ceiling applies on the cycle's primary task class is open. ADR-011's "default-not-ceiling" reading from essay 003 finds an operational instantiation here: OI-MAS (arXiv:2601.04861) routes individual decisions confidence-gated to a more capable tier when the cheap-tier confidence is below threshold, achieving 17–78 percent cost reduction with +12.88 percent accuracy. Cheap by default, escalate when aggregation demands exceed cheap-tier capability.

Generative topology construction is potent but infrastructure-heavy. DyTopo (per-round descriptor emission for runtime topology synthesis) achieves +6.2 percentage points; HERA (reward-guided experience retrieval for topology selection, arXiv:2604.00901, 2026) achieves +38.69 percent. Both outperform static baselines substantially but require infrastructure beyond the closed five-tool surface — embedding models, reward signals — that exceed llm-orc's current commitments without architectural extension. DyTopo's per-round descriptor emission is the lightest-weight published approach.

Typed contracts are the cycle's novel empirical territory. The MCP specification (November 2025) mandated structured output schemas; PydanticAI enforces them at the framework level; Arazzo provides workflow-level contracts. **No peer-reviewed paper measures the reliability delta from typed-contract enforcement.** llm-orc's Composition Validator plus Calibration Gate combination would produce novel empirical evidence in this territory. The tension between typed contracts and generative composition is real but resolved by HERA's pattern: member-interface schemas stable, topology generation operates above that boundary.

Calibration-gated routing has direct literature support. ReDAct (arXiv:2604.07036, 2026) shows that deferring only 15 percent of decisions to a large model matches full large-model quality. Chuang et al. (arXiv:2502.04428, 2025), across 1,500-plus settings, establish that uncertainty-quantification method choice dominates threshold choice in calibration-gated routing — a direct design recommendation for ADR-007's Calibration Gate implementation.

Retrieval-grounded selection beats LLM-judgment routing. R3AG (arXiv:2604.22849, 2026) and HERA both validate that retrieval-grounded routing outperforms both static routing and LLM-judgment routing. HERA's reward-guided experience retrieval is the closest published implementation of "stock library plus retrieval-grounded selection" — the user's research-entry design intuition for the cycle. The performance improvement is contingent on Plexus enablement (currently deferred WP-K). In Plexus-absent mode, this shape degrades to static stock-library selection.

For the script-models layer, the cycle's empirical anchor (Spike C's 3-of-3 cross-file verification) finds direct literature support. LLMCompiler (arXiv:2312.04511, ICML 2024) delivers 3.7× latency speedup and 6.7× cost reduction over ReAct by separating dependency-graph planning (LLM) from parallel execution (deterministic). Agentic Compilation (arXiv:2604.09718, 2026) reduces per-execution cost from approximately $150 to under $0.10 over 500 runs of a 5-step workflow. The amenability condition is stable, repetitive, structure-amenable workflows. For Cycle 4's territory, link checking and model-profile validation compile cleanly; semantic documentation analysis does not. Compiled AI's hybrid mode — deterministic compilation for structure-amenable verification plus runtime LLM for semantic analysis — is the pattern that fits the cycle's mixed task class.

The strongest published support for Spike A3's script-member-alongside-LLM pattern came from Wisdom and Delusion of LLM Ensembles (arXiv:2510.21513, October 2025): CrossHair counterexample feedback embedded in a code-generation ensemble prevents LLM consensus from overriding tool evidence. The mechanism is categorical, not probabilistic — deterministic outputs cannot be argued away by LLM consensus. This is the published support for Spike A3's pattern, and it surfaces an open question the literature does not address: what happens when deterministic tool output conflicts with LLM consensus? No paper studies this failure mode directly.

For the orchestrator-routing layer, pre-specifiable routing exceeds LLM-judgment routing across the literature scanned (CAAF, LLMCompiler, MetaAgent). Gu (arXiv:2603.11011, March 2026) shows routing based on pre-computed capability profiles and coordination-risk priors is effective without real-time capability judgment — directly applicable to the cheap-cloud-orchestrator constraint. Bounded autonomy with schema-level enforcement is convergent deployment practice, aligned with ADR-008's per-session Autonomy Levels architecture. MetaAgent (arXiv:2507.22606, ICML 2025) demonstrates auto-designed FSM topologies with state traceback for error recovery — a direct class (c) decomposition pattern.

## Long-Horizon Reliability Infrastructure

The third lit-review covered long-horizon reliability *infrastructure* as a coherent design surface — the externalization patterns, initializer-then-resume mechanics, calibration-gated cross-layer composition primitives, and per-role configurability patterns that the prior lit-review had identified as cross-cutting concerns. The review's load-bearing finding is its architectural verdict: long-horizon reliability infrastructure is **operationalizable within the existing four-layer architecture** as elaborations of existing module responsibilities, not as a fifth layer — *with one explicit caveat*: whether the responsibility concentration this elaboration produces (Session Registry absorbing initializer-then-resume schema plus write-gate validation plus append-only persistence; Calibration Gate absorbing trajectory-level calibration alongside post-hoc promotion) is acceptable is itself an architectural design choice the literature does not resolve for the cycle. The "operationalizable within existing layers" reading is conservative — the alternative reading (the cycle's design methods require modules of substantially expanded scope, warranting evaluation of whether dedicated infrastructure modules would be architecturally cleaner) is a DECIDE-phase question rather than a settled one.

The Anthropic initializer schema is the canonical reference for externalized state. The pattern uses three artifacts: `feature_list.json` with 200-plus entries and a monotonic boolean `passes` field that enforces structural non-regression at the schema level, not through agent compliance; `claude-progress.txt` as an append-only free-text log; `init.sh` as the deterministic environment bootstrap. The monotonicity constraint is the critical design element — non-regression is enforced at the schema level, not through agent compliance. This is a class (a)/(c) hybrid in the intervention-class taxonomy: structural override at the artifact-schema level.

The three-tier loading pattern (hot memory always-loaded constitution; domain specialist agents per-task; cold memory on-demand retrieval) has empirical validation across 283 sessions on a 108K-line codebase with 16,522 agent turns (arXiv:2602.20478). MemMachine's nucleus expansion (arXiv:2604.04853) reaches 93.0 percent on LongMemEvalS at 80 percent fewer tokens than Mem0 — and produces a directly-actionable design implication: retrieval-stage tuning contributes 4× more improvement than ingestion-stage tuning. Investment should target retrieval quality, not ingestion schema. Plain filesystem scores 74 percent on MemoryAgentBench, outperforming naive vector-store libraries on the same benchmark, which produces a complementary design implication: don't invest in retrieval infrastructure before simple structured-artifact approaches saturate.

Mnemonic sovereignty (arXiv:2604.16548) identifies six lifecycle phases of agent memory with distinct failure modes. Append-only JSONL supports auditability and session operations (resume, fork, rewind) but creates rollback limitations and a memory-poisoning attack surface. Write-gate validation is recommended in the literature but not operationalized in any reviewed system — this is potential Cycle 4-or-later territory.

For initializer-then-resume mechanics, the Anthropic schema specifies six fixed steps in session-start: confirm working directory, read progress log, read git history, consult feature list, run init.sh, select one feature. Full-read is used for small structured artifacts; governed retrieval is used for large knowledge stores. The two approaches are complementary, not competing. OpenDev's Planner subagent enforces planning at the schema level — write tools are simply absent from the planner's tool surface, not a prompt instruction to avoid writing. This avoids the alignment tax at small model sizes (the 12-point accuracy drop from constrained decoding at 8B); schema gating imposes no decoding overhead.

Claude Code's five-layer compaction pipeline is operationally specified in detail. Layer 0 persists tool results larger than 50K characters to disk and injects a 2KB preview plus path. Layer 1 deletes old cache entries without invalidating the prefix using the cache_edits API. Layer 2 clears idle-expired tool results after a 60+-minute gap. Layer 3 produces a free summary via continuously-maintained session notes — zero LLM cost, nine-section template (current state, tasks, files, workflow, errors, learnings, worklog), 12K-token cap. Layer 4 generates an LLM semantic summary as last resort with a circuit-breaker after three failures. The 250,000 API calls per day waste documented from pre-circuit-breaker Layer 4 failures (Claude Code's specific pre-circuit-breaker failure history, not a general finding — but motivationally instructive for any system planning LLM-based compaction) validates the cheapest-first ordering. Layer 3's zero-cost-notes pattern is directly portable to llm-orc's Conversation Compaction module — the nine-section template is adoptable as-is.

A notable contrast: Cognition Labs explicitly states Devin does not maintain cross-session memory as of mid-2025. The most prominent deployed coding agent does not implement the pattern the literature converges on as best practice. Conversely, Confucius Code Agent achieves the highest reviewed result on SWE-Bench-Pro — 59 percent Resolve@1 per the paper's abstract (a results figure in the same paper shows 54.3 percent; the discrepancy reflects version differences in the paper's reporting) — with persistent cross-session note-taking as the stated mechanism (arXiv:2512.10398). The contrast signals that the cycle's design methods should treat externalized state as primary, not as optional optimization.

For calibration-gated cross-layer composition, the Dual-Process AUQ framework (arXiv:2601.15703) is training-free: System 1 propagates verbalized confidence through attention as a soft constraint; System 2 applies a binary gate at confidence thresholds 0.8–1.0. Low confidence triggers reflection rather than blocking, achieving +10.7 percent on ALFWorld and +13.6 percent on WebShop. Trajectory-level calibration via HTC (arXiv:2601.15778) extracts process-level features across entire trajectories — more informative than output-level calibration for long-horizon decisions, with cross-domain transfer validated without retraining. **Calibration mechanisms are architecturally portable across agent frameworks.** OI-MAS (arXiv:2601.04861) is the closest published analogue to an in-process Calibration Gate: higher confidence penalizes expensive model choices, lower confidence triggers tier escalation, achieving 17–78 percent cost reduction with +12.88 percent accuracy.

The literature does not yet implement calibration as a *cross-layer* primitive — L0 ensemble outputs gating L1 dispatch decisions. The components exist (AUQ, HTC, OI-MAS); the composition does not. **This is the cycle's most novel territory.** ADR-007's current Calibration Gate is post-hoc output-level calibration: last-N positive signals trigger promotion. The literature points toward layering in-process trajectory-level calibration alongside it. A fully realized gate would combine both: post-hoc promotion tracking (ADR-007 as implemented) plus in-process confidence gating (AUQ/OI-MAS pattern) for within-session dispatch decisions.

Making calibration a cross-layer primitive requires a read-only signal channel from L0 (ensemble outputs) to L1 (Calibration Gate dispatch decisions) — which the current layering rule prohibits ("edges never upward"). **This is the one identified point where the four-layer architecture may need architectural extension** — not a fifth layer, but an explicit signal channel upward from L0 to L1 within the existing layer structure. Whether this violates the spirit of the layering rule is a DECIDE-phase question.

For per-role configurability, the literature's empirical evidence is consistent and consequential. Heterogeneous role-staffing beats homogeneous, repeatedly. SC-MAS achieves +3.35 percent accuracy and -15.38 percent cost on MMLU. MasRouter delivers +1.8 to +8.2 percent accuracy on MBPP and -52 percent cost on HumanEval (separate benchmarks). OI-MAS provides +12.88 percent accuracy at -17 to -78 percent cost. The mechanism, named explicitly: capability saturation. Homogeneous systems over-allocate expensive capability to tasks that don't require it. The Topaz eight-skill taxonomy (code generation, tool use, mathematical reasoning, logical reasoning, factual knowledge, writing quality, instruction following, summarization) is directly adoptable as a role-profiling vocabulary. The key result, quoted from Topaz's authors: "Efficiency gains stemmed from capability saturation rather than hidden quality loss" — routing cheaper models to tasks where expensive models add no marginal value is *not* a quality compromise.

External memory couples with per-role configurability in the literature: small models cannot maintain coordination context independently, and external memory is what makes small-model role-staffing viable. The two design surfaces are not independent choices.

## The Behavioral Spike — Single-Iteration Validation and Refinements

Wave 3.A ran a behavioral spike grounding the cycle's synthesis at the deployment shape. The spike question: can the cheap-orchestrator + ensemble pattern, dispatched via `llm-orc serve`'s direct-API `invoke_ensemble` tool surface (avoiding Cycle 3 Spike D's opencode-CLI shell-subprocess stall), drive one bounded RDD-research-loop iteration to a recognizable completion? Three trials at the cheap-cloud tier (MiniMax M25 Free via OpenCode Zen, free-tier per the corpus's spike policy) using a small spike-only ensemble dispatched through `llm-orc serve` direct API.

The dispatch path validated. Two of three trials produced clean dispatches at 21–32 seconds wall-clock with no stall — direct contrast with Cycle 3 Spike D's 14-minute-to-14-hour opencode-CLI hangs. The closed five-tool surface (ADR-003) sufficed: the orchestrator did not request any tool outside it. Long-horizon session continuity was *not* a binding constraint within one iteration; context-rot symptoms were single-turn (harness compression), not multi-turn (accumulation). Trial 1 produced positive evidence on a precondition for Cycle 3 grounding action 2's autonomous-routing question: the orchestrator dispatched `spike-cycle4-research-loop` correctly without explicit staging on a one-fixture, one-ensemble-in-library setup. The reformulated Sub-Q6 transfer-test asks whether context growth degrades ensemble-routing judgment at the session lengths the North-Star benchmark requires; Trial 1 answers whether the path *fires at all* (yes, on this configuration), not whether routing judgment degrades as context accumulates across iterations. These are different questions. Trial 1 establishes a necessary precondition for the transfer-test (the path must work before degradation can be observed) but does not itself constitute partial evidence on the transfer question. The Sub-Q6 transfer-test remains entirely open at cycle close.

Two negative findings refined the cycle's synthesis substantively. Trial 2 — explicitly-instructed dispatch — surfaced a specificity-loss artifact: the orchestrator claimed "verbatim from ensemble return" what was actually the Result Summarizer Harness's compressed blob. The Harness re-instantiates Cycle 2 Spike A's specificity-loss mechanism on the orchestrator's view of ensemble outputs. The mechanism that Spike A documented at the two-stage cascade-collapse level reproduces at the harness-interposition level on a *single-iteration* trial. This empirically validates the cycle's prior literature-derived flag on the Harness as a candidate reliability risk: the issue is not theoretical.

Trial 3 — diagnostic — produced a confabulation pattern: the cheap-cloud model wrote prose claiming a tool call had occurred without emitting any tool-call structure. The exact text: "The tool call has been made and the result is displayed above as a `role:tool` observation." Zero artifacts existed. The class of failure is *not* novel — Wave 1.A's lit-review documents that "models like Ollama 7B/14B often hallucinate tool calls as plain text after the first interaction" as a practitioner-documented finding for small local models. The novel element is observation at the cheap-cloud tier on a substantially more capable model. *Trial 3 used an adversarial diagnostic prompt* ("Output nothing else. Just the tool call. No prose, no explanation, no narration") rather than conditions representative of operational agentic sessions; the 1-of-3 confabulation rate is confounded by the prompt design and should not be used to calibrate the urgency of the runtime-side guard. The failure pattern is qualitatively distinct from the failure modes the cycle's prior literature scan covered for *cloud* tiers (judgment-decay, attention drift, instruction recency bias).

A methodological finding emerged from the spike's analysis. The chat-completions response body returns the *final* assistant message only; the tool-call trajectory has already been consumed pre-final. The spike's initial reading saw `n_tool_calls: 0` in the response body and concluded all three trials had failed. The artifact-directory cross-check produced the actual trajectory. Future spikes should not read response-body-only as evidence of dispatch absence; the trajectory-inference-via-artifact-directory-cross-check approach is now a documented spike methodological pattern.

The spike's findings refined the cycle's synthesis in three specific places. First, the empirical reproduction of specificity-loss at the orchestrator's view of ensemble outputs strengthens the cycle's flag on the Result Summarizer Harness from literature-suggestion to empirical-finding — the issue manifests in a single iteration, not only at multi-stage scale. Second, phantom tool-call confabulation is a new failure-mode hook the literature did not cover and adds a candidate intervention class — runtime-side structural validation of orchestrator claims of tool calls. Third, the autonomous-routing positive observation (N=1, one-fixture, one-ensemble-in-library) establishes a *precondition* for Cycle 3 grounding action 2's evidence gap — the path fires; whether routing judgment degrades under context growth (the Sub-Q6 transfer-test) remains entirely open and is Cycle 5+ territory. Fourth, and importantly for honest scope-of-claim, the cycle's claims on long-horizon reliability infrastructure remain literature-grounded but not cycle-empirically-grounded — the spike is single-iteration by design and could not test multi-iteration accumulation.

## The Composite Framing: Layer-Conditional Composition

The Wave 1+2+3 findings converge into a coherent design-method posture for the cheap-orchestrator + ensemble pattern under the North-Star benchmark. The composite framing has four stacked elements.

**Layer-conditional composition shape selection** is the primary frame. Different shapes are appropriate at different architectural layers given the task class at each layer. Cross-file verification: script-models layer is the load-bearing mechanism (Cycle 3 Spike C's 3-of-3 versus 0-of-2 evidence), with the ensemble as wrapper. Documentation review with per-recommendation specificity: ensemble composition is the load-bearing mechanism — MARG-style concatenation plus heterogeneity, no-collapse-summarization (Cycle 2 Spike A3 evidence, strengthened by DeliberationBench's 6× selection-versus-deliberation gap). Hybrid deployment economics: orchestrator routing plus tool dispatch is the load-bearing mechanism (Cycle 1 CAP-9). Long-horizon session continuity: cross-cutting infrastructure operationalized within existing layers (Wave 2.B architectural verdict; not a new layer).

**Pre-specifiable routing as primary design axis within each layer** is the second frame element. Pre-specifiable routing exceeds LLM-judgment routing across CAAF, LLMCompiler, MetaAgent, and Gu (2603.11011) — the literature provides the genuine support for the comparison. Wave 3.A's spike validates the dispatch path operates end-to-end on one fixture with one ensemble in the library (eliminating routing ambiguity as a confound) but does not separately establish pre-specifiable routing as superior to LLM-judgment routing — the spike's autonomous dispatch fired correctly precisely because there was only one research-shaped free-tier ensemble to route to; routing-among-multiple-research-ensembles is not tested. Class (c) decomposition is the dominant intervention class for continuous routing reliability under context growth (Wave 1.A). The cycle's existing class (c) primitive — ADR-003's closed five-tool surface — is structurally a class (c) decomposition. The orchestrator's job is correct-dispatch-at-known-points; real-time capability judgment is replaced by task-aware delegation cues (Gu) operating on pre-computed capability profiles.

**Typed contracts as universal interface discipline** is the third frame element. The Composition Validator pattern (currently in L1) applied at every layer boundary. The cycle's empirical territory is novel here — no peer-reviewed paper measures the reliability delta from typed-contract enforcement. The architecture's Composition Validator + Calibration Gate combination would produce evidence the literature does not yet have. The tension between typed contracts and generative composition is real but resolved by HERA's pattern: member-interface schemas remain stable; topology generation operates above that boundary.

**Experience accumulation as the progression arc** is the fourth frame element. The progression: stock-library → retrieval-grounded → Plexus-store-driven. HERA's reward-guided experience retrieval is the closest published implementation of "stock library plus retrieval-grounded selection." Performance gains are contingent on Plexus enablement. In Plexus-absent mode, the architecture degrades to static stock-library selection — acceptable as default but caps the capability ceiling.

The four frame elements compose with one consequential tension to surface: Framing B (pre-specifiable routing as primary axis) and Framing D (experience-accumulation arc toward HERA-style retrieval-grounded selection) represent a design-axis tradeoff rather than a clean stack. HERA's 38.69-percent improvement over static routing baselines is precisely the value of *superseding* pre-specified routing, not augmenting it. Framing B holds in Plexus-absent mode (the cycle's current deployment state); Framing D partially supersedes B as Plexus activation enables retrieval-grounded selection. The two are not independent — Plexus activation erodes B's centrality. Layer-conditional shape selection (A) is operationalized as elaborations of the existing four-layer architecture's modules; pre-specifiable-versus-generative (B) is the primary design axis within each layer *for the current Plexus-absent deployment*; typed-contract interfaces (C) unify the layer boundaries; the experience-accumulation arc enabled by Plexus activation (D) is the progression *direction* that weakens B's necessity rather than a stable ceiling. Together they constitute the cycle's design-method posture, with the B↔D tradeoff named as future-state evolution rather than fixed architectural axis.

The framing addresses each of the cycle's six sub-questions directly. Sub-Q1 (long-session capability decomposition): failure modes settled at the macro level (Khanal's 76 → 52 percent; SWE-EVO's 47.8-point drop; SWE-Bench Pro's 23 percent ceiling); cross-cutting infrastructure (externalized state, initializer-then-resume, per-role configurability, in-process calibration) is the design surface — operationalizable within existing layers. Sub-Q2 (mechanism attribution): layer-conditional per the analytical decomposition and Cycle 3 evidence; the unifying frame (composition of components with different error distributions) holds with layer attribution depending on task class. Sub-Q3 (load-bearing layer): no single layer carries the value across all task classes; the largest framing shift from Wave 2.B is that long-horizon infrastructure operationalizes within existing layers rather than as a new layer; one architectural-extension candidate is the upward read-only signal channel for cross-layer calibration. Sub-Q4 (composition-shape design space): plural and layer-conditional. Sub-Q5 (RDD-cycle benchmark decomposition): three decision-class clusters (Cluster 1 specialist-dispatch / Cluster 2 continuous-routing / Cluster 3 conversational), with class (a) harness patterns appropriate at Cluster 1's finitely-occurring decisions and class (c) decomposition + class (b) governed retrieval at Cluster 2's continuous routing. Sub-Q6 (intervention class): RDD's structural-hook finding's transfer to ensemble routing is partially supported (shared mechanism with phase-transition reliability) but the decision-class differs; Spike A3's class (a) wrapper does not generalize to ensemble routing; the right intervention class is class (c) decomposition (pre-specifiable routing, schema-level enforcement) and class (b) governed retrieval (HERA-style); calibration-gated cross-layer composition (operationalized via ADR candidate #6's upward L0→L1 read-only signal channel) is the cycle's most novel territory — the components exist (AUQ for in-process confidence, HTC for trajectory features, OI-MAS for tier-escalation) but their composition into a cross-layer primitive is unstudied in the literature.

## Seven ADR Candidates and the One Architectural-Extension Question

The cycle's research surfaces seven concrete design decisions as candidate ADRs. Six are elaborations or extensions of existing architectural commitments; one amends the layering rule.

**ADR candidate #1 — Conversation Compaction adopts Claude Code's five-layer pattern.** The Orchestrator Runtime (L2) elaborates its existing Conversation Compaction awareness with a five-layer compaction pipeline: persist-large-tool-results, cache-edit, idle-expiry, free-summary-via-session-notes, LLM-semantic-summary-as-last-resort with circuit-breaker. Layer 3's zero-cost-notes nine-section template is directly adoptable. The 250,000-API-calls-per-day waste finding from pre-circuit-breaker Layer 4 failures validates cheapest-first ordering.

**ADR candidate #2 — Session Registry adopts initializer-then-resume schema.** The Session Registry (L3) takes on the additive responsibility of structured-handoff artifacts: a feature-list-with-monotonic-passes schema (structural non-regression at the schema level, not prompt compliance), an append-only progress log, an init-sh-style deterministic environment bootstrap, and write-gate validation for memory-poisoning protection. The Anthropic schema is the canonical reference; the design decision is which artifacts to require versus support, and what the write-gate validation surface looks like.

**ADR candidate #3 — Calibration Gate extension.** The Calibration Gate (L1) significantly elaborates ADR-007 with in-process trajectory-level calibration alongside the existing post-hoc promotion tracking. HTC trajectory-level features (process-level information across entire trajectories, more informative than output-level for long-horizon decisions) and AUQ-style soft-then-binary gating (System 1 attention-propagated confidence; System 2 binary threshold at 0.8–1.0; low confidence triggers reflection not blocking). ADR-007's Open Question #5 — "what counts as a quality signal" — is filled in.

**ADR candidate #4 — Per-role tier-escalation router.** The Tool Dispatch (L2) elaborates its existing interposition logic with a confidence-gated tier-escalation pattern via OI-MAS — cheap model by default, escalate to a more capable tier when confidence is below threshold. Topaz's eight-skill taxonomy is the role-profiling vocabulary. ADR-011's "default-not-ceiling" reading from essay 003 is operationally instantiated. This is consistent with ADR-011's existing rule: ADR-011 already says tiered behavior is expressible as ensemble-composition; OI-MAS at Tool Dispatch is an implementation of ADR-011, not an amendment. ADR-011's session-boundary-event constraint on the orchestrator's own LLM is preserved (the per-task router escalates the dispatched task, not the orchestrator).

**ADR candidate #5 — Result Summarizer Harness reconsideration.** The Tool Dispatch (L2) interposition logic for the Harness adopts a *summary-with-pointer* mode (return summary plus persistent reference to the full ensemble output, accessible via `query_knowledge` or a new pointer-following primitive) or *skip-summarization-when-short* (gate activation by output-length / content-density threshold). Empirically *suggested* by Wave 3.A Trial 2 (one trial on a single fixture; the ensemble synthesizer produced approximately 600 characters yet was summarized anyway, and the orchestrator quoted "verbatim" what was actually the summarizer's compressed blob) and by DeliberationBench's 6× selection-versus-deliberation gap at the literature level. The single-trial empirical motivation is below the evidentiary threshold for amending ADR-004's mandatory commitment in DECIDE — a targeted follow-up spike on diverse output sizes and ensemble configurations is warranted before architectural amendment. Spike A's documented mechanism (two-stage cascade-collapse in multi-agent documentation review) is a different architectural position (reviewer-to-aggregator stage in a multi-trial documentation review) than the harness-interposition stage observed in Wave 3.A's single trial; bridging them requires the additional spike. The Harness's interposition value (preventing oversize outputs from dominating orchestrator context) is preserved for cases where summarization adds compression cost with compression benefit; selective skip preserves specificity for cases where it does not. Candidate amendment territory pending the targeted spike.

**ADR candidate #6 — Upward L0→L1 read-only signal channel.** Cross-layer calibration (L0 ensemble signals gating L1 dispatch decisions) requires an upward read-only signal path — which the architecture's layering rule prohibits ("edges never upward," stated in `system-design.md` and verified by FC-2 / FC-3; ADR-002 establishes the four-layer architecture but the explicit edge-direction rule lives in the system design). This ADR amends the layering rule to permit a specific exception: a read-only signal channel from L0 to L1 for the cross-layer Calibration Gate's input. The amendment is small in scope but consequential in principle. Justification: ADR-002 already acknowledges "two code paths to maintain for orchestrator behavior: with-Plexus and without-Plexus" as a negative-consequence cost paid for the architecture's value; the upward-signal-channel amendment adds a similarly-acknowledged cost paid for cross-layer calibration's value. The amendment's scope is read-only (no upward writes; the layering rule's structural integrity for write paths is preserved) and signal-channel-specific (calibration only; not a general upward import permission).

**Cycle/scale risk and required bounding mechanisms (gate-conversation refinement, 2026-05-04):** The signal path creates a feedback shape — Calibration Gate (L1) gates dispatch → Ensemble Engine (L0) runs → signals flow back to L1 → Calibration Gate uses signals for the next dispatch. Feedback paths can compound bias, and the cycle's literature evidence makes this risk concrete: Khanal et al. (arXiv:2603.29231) found *universal non-improvement* from episodic memory augmentation across all ten tested models at long horizons (six models showed negative effects, four were neutral, none improved); CAAF (arXiv:2604.17025) flagged that "apparent LLM reliability in safety-critical domains is often a prompt engineering artifact" — calibration signals are themselves model outputs, and gating on biased outputs entrenches bias; Li et al. (ICLR 2026) documented the trigger-vulnerability finding that injecting *objective* context into a debate accelerates polarization rather than moderating it (the directional caution transfers from debate-shape coordination to feedback-shape signal flow but the original study's domain is multi-agent debate, not unidirectional read-only calibration channels — Cycle 2's lit-review explicitly flagged this transfer as empirically open). ADR candidate #6's drafting in DECIDE must therefore specify five bounding mechanisms as load-bearing design surface, not decoration: **(a) fresh-context isolation in the consumer** (the calibration consumer reads signals in a context that does not carry prior signal forward — the architectural-isolation pattern the methodology itself uses for audit subagents); **(b) time-decay windowing** to bound the bias-compounding horizon (only recent signals influence current decisions; old signals decay); **(c) categorical anchors via deterministic-tool-output where ensemble composition includes script-model members** (Wisdom and Delusion of LLM Ensembles, arXiv:2510.21513, on CrossHair counterexample feedback — deterministic outputs cannot be argued away by LLM consensus, so the feedback loop cannot drift on probabilistic noise; this mechanism is *ensemble-composition-conditional* — only available where the ensemble has script-model slots; LLM-only ensembles cannot use it and DECIDE must specify whether the full five-mechanism set is required or whether a documented subset satisfies the conditionality for LLM-only configurations); **(d) periodic out-of-band audit dispatch** to check whether the calibration system itself is drifting (the susceptibility-snapshot pattern from RDD methodology, applied to architectural calibration rather than to research framing); **(e) read-only structural validation at the consumer** (CAAF's class-(a) pattern — validate schema and type of the signal before acting on its content; catches malformed signals before they influence decisions). The amendment to ADR-002's layering rule is conditional on these bounding mechanisms being load-bearing in the implementation. The argument-audit and conformance-audit on ADR candidate #6 in DECIDE must pressure-test whether each bounding mechanism is genuinely operationalized rather than decorative; absent operational bounding mechanisms, the precedent erodes the layering rule's discipline rather than carving a contained exception. **Whether the cycle/scale risk can be bounded effectively is the load-bearing question DECIDE must answer.**

**ADR candidate #7 — Tool-call structural validation.** The Tool Dispatch (L2) adds a class (a) guard: validate that orchestrator claims of tool calls correspond to actual tool-call structures emitted; reject mismatches as malformed-batch consistent with the existing typed-error path for per-model tool-calling rejection. Empirically motivated by Wave 3.A Trial 3 — phantom tool-call confabulation. The recent commit `9f86d0b feat: raise typed error when provider rejects tool calling per-model` is the structurally-similar precedent in the codebase.

The seven ADR candidates do not share a single epistemic profile and DECIDE-phase deliberation should reflect the distinction. **ADR candidates #1 (Conversation Compaction five-layer pipeline) and #2 (Session Registry initializer-then-resume schema) are direct adoptions of canonical external patterns with reference implementations** (Claude Code's compaction pipeline; Anthropic's `feature_list.json` / `claude-progress.txt` / `init.sh` schema). The DECIDE work on these is largely adoption-decision discipline — verifying applicability to llm-orc's deployment shape, identifying llm-orc-specific deviations, and writing the ADR with full-credit attribution to the source pattern. **ADR candidates #3 (Calibration Gate trajectory-level extension), #5 (Result Summarizer Harness reconsideration, pending the targeted spike), #6 (upward L0→L1 signal channel — the layering-rule amendment), and #7 (tool-call structural validation guard) involve novel architectural territory** where the literature provides components but not the composition. The DECIDE work on these is genuine architectural deliberation: alternatives must be examined, scope conditions named, conformance-audit against existing code surfacing real divergences. **ADR candidate #4 (per-role tier-escalation router) sits between** — the OI-MAS pattern is well-documented externally but its operationalization within ADR-011's session-boundary-event constraint requires deliberation. The DECIDE phase deliberation budget should concentrate on the four novel-architectural candidates (#3, #5, #6, #7) plus the bridge case (#4); the two adoption candidates (#1, #2) warrant lighter argument-audit and conformance-scan attention but heavier deviations-from-source documentation. ADR candidate #6 in particular benefits from DECIDE's scenario-writing and conformance-audit-against-existing-code discipline: the upward-signal-channel amendment touches FC-2 (static import check) and FC-3 (cycle detection) in the existing test architecture, and the amendment's permitted-versus-prohibited boundary needs scenarios.

## Implications for the Architecture

The research's findings have specific implications for the architecture's existing commitments without forcing wholesale changes.

ADR-002 (four-layer architecture) holds, with one amendment candidate (the upward signal channel for cross-layer calibration). The architecture's value proposition is preserved: Layers 1–3 remain the baseline; Layer 4 (Plexus) remains the additive upgrade. The "two code paths to maintain" cost the ADR already acknowledges expands to "two code paths plus one structural exception" — a contained scope.

ADR-003 (closed five-tool internal action surface) is *strengthened* by Wave 3.A's empirical evidence. The closed surface sufficed for autonomous routing on the cycle's spike fixture. The pre-specifiable routing finding from CAAF, LLMCompiler, MetaAgent, and Gu's task-aware delegation cues all support the closed-tool-surface design. ADR candidates #4 (per-role router) and #7 (tool-call structural validation) elaborate Tool Dispatch; ADR candidate #5 (Harness reconsideration) amends ADR-004's mandatory-summarization commitment in scope.

ADR-004 (Result summarization mandatory) is the candidate amendment from ADR candidate #5. The cycle's evidence — Wave 3.A Trial 2's single-trial specificity-loss observation on a single fixture (a 600-character ensemble output that was summarized anyway) plus DeliberationBench's 6× selection-vs-deliberation gap at the literature level — *suggests* that "mandatory" might become "default-with-conditional-skip" but is below the evidentiary threshold for amending the existing ADR commitment. A targeted follow-up spike on diverse output sizes and ensemble configurations is warranted before DECIDE-phase amendment proceeds; treat ADR candidate #5 as candidate amendment territory pending that spike.

ADR-007 (Calibration Gate for composed ensembles) is the candidate elaboration from ADR candidate #3. The current ADR fills in Open Question #5 (what counts as a quality signal) with HTC trajectory-level features and AUQ confidence. The current ADR's scope (calibration of *composed* ensembles' first-N invocations) extends to in-process calibration of dispatch decisions across sessions. The cross-layer extension (ADR candidate #6) is the architectural-amendment piece.

ADR-008 (per-session autonomy levels) holds, with the pre-specifiable-routing literature providing strengthening evidence. Bounded autonomy with schema-level enforcement is convergent deployment practice. The cycle's autonomous-routing positive observation (Wave 3.A Trial 1) supports cheap-cloud-tier autonomous dispatch within the closed five-tool surface — autonomy gating remains the right cross-cutting policy.

ADR-011 (orchestrator LLM is a Model Profile) holds, with essay 003's "default-not-ceiling" reading operationally instantiated by ADR candidate #4 (per-role router via OI-MAS). The session-boundary constraint on the orchestrator's own LLM is preserved; the per-task router escalates dispatched-task LLM tier without switching the orchestrator's. The "tiered behavior expressible as ensemble-composition" framing of ADR-011 is consistent with the OI-MAS pattern.

The system-design's Conversation Compaction module (mentioned in §"Modules — brief" but not yet specified in detail) gains an empirically-motivated specification candidate from ADR candidate #1 (Claude Code's five-layer pattern). The Session Registry's session-state responsibility expands from the existing "identifies and continues a multi-request Session" to include initializer-then-resume artifact management per ADR candidate #2.

The Plexus Adapter remains optional with no-op fallbacks (ADR-002 baseline). The cycle's findings on retrieval-grounded selection (HERA pattern; ADR candidate #4's tier-escalation cues; the experience-accumulation arc) are *contingent on Plexus enablement* — Plexus-absent mode preserves the architecture's value but caps the capability ceiling at static stock-library selection. WP-K's deferral remains operative; the cycle does not motivate accelerated Plexus activation but does sharpen the capability gap Plexus closes.

## Open Questions and Scope-of-Claim

The cycle's findings carry several explicit scope conditions that the DECIDE phase and downstream cycles should attend to.

**Long-horizon reliability infrastructure claims are literature-grounded, not cycle-empirically-grounded.** Wave 3.A is single-iteration by design; multi-iteration accumulation effects (where context-rot, working-memory degradation, and retrieval-quality dependencies most operate) were not tested. The cycle's claims on externalized state, initializer-then-resume, calibration-gated cross-layer composition, and per-role configurability are supported by converged literature best-practice (Anthropic, OpenDev, HELM, Claude Code, Confucius Code Agent, MemMachine) but await direct cycle empirical validation. Cycle 5+ territory.

**Autonomous routing remains partially validated.** Wave 3.A Trial 1's positive observation is N=1 on a single fixture at the cheap-cloud tier. Cycle 3 grounding action 2's evidence gap is partially closed but not fully. Multi-iteration scale, fixture diversity (different RDD-cycle phases, different research questions), and N>1 trials are required before autonomous routing can be claimed reliably. The cycle's pre-specifiable-routing recommendations (class (c) decomposition; schema-level enforcement) are consistent regardless of whether autonomous routing reaches full reliability — they reduce the surface where it matters.

**The cycle did not test frontier-tier comparison directly.** Cycle 3 grounding action 3's frontier-baseline-with-matched-information-access question was not addressed in Wave 3.A — frontier comparison was deferred until the multi-iteration spike per the spike-runner's recommendation. The MOP-paradox finding from the literature (frontier models exhibit highest meltdown rates on long-horizon) is suggestive that the cycle's framing of cheap-cloud-orchestrator + good-architecture as the primary path is well-calibrated, but direct comparison-baseline evidence at the cycle's deployment shape is Cycle 5+ territory.

**The Self-MoA versus heterogeneity tension remains task-class-conditional.** Self-MoA's finding (sampling from a single best-performing model beats cross-family mixing on instruction-following) and Spike A3's heterogeneity-uncorrelated-errors finding (cross-family produces uncorrelated findings with low overlap on documentation review) coexist as task-class-dependent results. The cycle's task class (agentic verification, documentation review, code review) is on the heterogeneity side. Whether the cycle's design methods generalize to task classes Self-MoA addresses (instruction-following with well-defined questions) is open.

**Typed-contract reliability delta is novel territory.** No peer-reviewed paper measures the reliability delta from typed-contract enforcement at the level the cycle's Composition Validator + Calibration Gate combination would exercise. The cycle would produce novel empirical evidence in this territory if the architectural-extension and ADR candidate #3 were operationalized.

**The phantom tool-call confabulation pattern is novel.** Wave 3.A Trial 3's failure mode (model writes prose claiming tool calls without emitting them) does not have a direct literature analogue in the cycle's lit-review scan. ADR candidate #7 addresses it with a runtime-side guard, but the failure mode's frequency, fixture-class dependence, and severity at multi-iteration scale are unknown.

**The 7B-14B reliability boundary is the load-bearing local-first finding.** Local models reliably handle single-file, bounded, stateless tasks; unreliably handle multi-step tool calling after context accumulation. The cycle's local-first commitment operationalizes within this boundary — the architecture's design methods (decompose complex decisions into many bounded stateless per-member tasks) align with the boundary's empirical shape. Whether the boundary shifts as open-weight models mature (Qwen3-Coder-Next is already at 70.6–71.3 percent on SWE-Bench Verified) is observation-territory rather than design-territory.

**The autonomous-routing-versus-pre-specified spectrum.** The cycle's framing prefers pre-specifiable routing for reliability, but pre-specifiability has a design cost — the routing logic must be captured at architecture time. Generative topology construction (DyTopo, HERA) sits at the other end of the spectrum and outperforms static baselines substantially. The cycle's resolution (HERA's pattern: stable member-interface schemas with topology generation above that boundary) is one valid choice; whether more aggressive generation patterns earn their cost on the cycle's deployment shape is open.

## Conclusion

The cheap-orchestrator + ensemble pattern's supported design methods are layer-conditional cross-layer composition shapes operationalized as elaborations of the existing four-layer architecture's modules, with pre-specifiable routing as the primary design axis within each layer, unified by typed-contract interfaces, and progressing along an experience-accumulation arc enabled by Plexus activation. The composite framing addresses the cycle's six sub-questions directly and surfaces seven concrete design decisions as candidate ADRs, six elaborating existing module responsibilities and one amending ADR-002's layering rule (the upward L0→L1 read-only signal channel for cross-layer calibration). Long-horizon reliability infrastructure — externalized structured state, initializer-then-resume, calibration-gated cross-layer composition, per-role model configurability — operationalizes within existing layers as elaborations of Session Registry, Orchestrator Runtime, Calibration Gate, and Tool Dispatch responsibilities. The architecture is in better structural shape than the cycle's gate finding initially implied; the gaps are operationalization gaps, not architectural-shape gaps.

The local-first commitment that distinguishes the cycle's deployment from the published literature's predominantly-frontier-tier evidence is operationalized as decompose-complex-decisions-into-many-bounded-stateless-per-member-tasks. Local models amplify deterministic and bounded-scope ensemble work where the 7B-14B reliability boundary holds; the cheap cloud orchestrator handles routing, summarization, and decisions requiring sustained multi-step reasoning; sessions are bound by externalized structured state. The MOP-paradox finding from the literature reframes the cycle's local-first commitment as well-calibrated rather than as a compromise: at the open-source models cohort Khanal et al. tested (DeepSeek V3, MiniMax M2.5, and others — not proprietary frontier), the most-capable models in the cohort exhibit highest meltdown rates on long-horizon, and cheap-orchestrator + good-architecture may be more reliable than frontier-of-cohort on the North-Star benchmark's task category. The architectural-reliability and capability-bounded-failure readings of the same data coexist; neither is settled. Extrapolation to proprietary frontier behavior is directionally plausible but not directly evidenced. The 24-point Terminal-Bench 2.0 gap between best open-weight and frontier is real, but the architecture's value is in the structural compensation it supplies rather than in the raw capability of any single tier — composition of components whose error distributions are different enough that the composition's coverage exceeds any single component's, *on tasks where no single best response is identifiable in advance* (verification, documentation review, code review where coverage matters). DeliberationBench's 6× selection-vs-deliberation gap and Self-MoA's same-family-beats-cross-family finding together indicate that for tasks where a best response *is* identifiable, selection quality may dominate over error-distribution heterogeneity as the mechanism. The unifying frame's scope is coverage-bound tasks; on identifiable-best-response tasks, the frame may not apply.

The behavioral spike validated the architecture's `invoke_ensemble` dispatch path at the cheap-cloud tier on a single bounded fixture and surfaced two refinements: the Result Summarizer Harness re-instantiates Cycle 2 Spike A's specificity-loss mechanism on the orchestrator's view of ensemble outputs (motivating ADR candidate #5's Harness reconsideration), and a phantom-tool-call confabulation pattern at the cheap-cloud tier (motivating ADR candidate #7's structural validation guard). The dispatch path itself works — autonomous routing fired correctly in one trial; the closed five-tool surface was sufficient when the orchestrator's task structure implied the dispatch (Trials 1–2 used operational-style prompts), though Trial 3's adversarial diagnostic prompt produced phantom-call confabulation suggesting the sufficiency claim is conditioned on prompt structure (whether operational agentic sessions consistently provide that conditioning is an open scope condition); long-horizon session continuity was not a binding constraint within one iteration. The cycle's claims on long-horizon reliability infrastructure remain literature-grounded but not cycle-empirically-grounded; multi-iteration validation is Cycle 5+ territory.

The cycle's research phase closes with the seven ADR candidates positioned for DECIDE-phase deliberation. The DECIDE phase's argument-audit, scenario-writing, and conformance-audit-against-existing-code disciplines will pressure-test each candidate at granularity the research-phase argument-audit on a single essay cannot reach. ADR candidate #6 (the upward signal channel amendment to ADR-002's layering rule) is the load-bearing architectural decision; the others are elaborations of existing module responsibilities. Together they constitute the cycle's design-formulation deliverable for the cheap-orchestrator + ensemble pattern: a concrete, layer-conditional, empirically-motivated design surface for long-session agentic coding under the local-first commitment.
