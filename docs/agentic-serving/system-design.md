## What this document is, who it's for, where to read what

This is the technical perspective on **agentic serving** — how llm-orc serves as the orchestrator backend for agentic coding tools (OpenCode, Roo Code, Cline, etc.) via OpenAI-compatible endpoints. It describes the four-layer system, the closed-set tool surface, the client-tool delegation boundary, and the supporting modules that govern Budget, Composition, Calibration, Autonomy, and the (optional) Plexus knowledge-graph integration.

**First-encounter readers** orient through the diagram, brief module summaries, and the Client Tool Surface Commitment below. **Agents constructing context** for code work read the companion file at [system-design.agents.md](./system-design.agents.md), which carries the full architectural drivers table, module decomposition (Owns/Depends/Inversion notes), responsibility matrix, dependency graph, integration contracts, fitness criteria, test architecture, and Appendix A per-phase susceptibility-snapshot briefs in the form best suited to that work.

The split between this artifact and `system-design.agents.md` is the **companion-file pattern** (ADR-084 Pattern B): a single human-facing surface here, with a parallel-sibling agent-context file at a predictable path. The diagram below retains its load-bearing role for human orientation; the dense reference material has been relocated rather than removed.

**Sequencing** lives in [roadmap.md](./roadmap.md). **Decisions** live in [decisions/](./decisions/). **Vocabulary** lives in [domain-model.md](./domain-model.md). **Behavior** lives in [scenarios.md](./scenarios.md) and [interaction-specs.md](./interaction-specs.md). If you are coming to this corpus with no prior context, read [ORIENTATION.md](./ORIENTATION.md) first.

**Cycle 7 loop-back — two coexisting surfaces on the Serving Layer** (the diagram above shows the v5.0 single-turn surface; the loop-back adds a parallel tool-driven surface). A **surface-mode discriminator** (Serving Layer; `len(request.tools) > 0`) routes each request:
- **Single-turn (no client tools)** → the **Dispatch Pipeline** (plan → dispatch → synthesize), the v5.0 surface — ADR-027 is scoped here by its ADR-033 partial-update header.
- **Multi-turn tool-driven (client tools present)** → **L2 Loop Driver** (layer-A control structure; decides each turn's next agentic step; delegates per-turn generation to a single capability ensemble — the callee) + **L2 Single-Step Enforcer** (batch-truncation; makes grounding a framework guarantee) + **L2 Artifact Bridge** (reads the server-side `SessionArtifactStore` deliverable and marshals it into tool-call content) + **L3 Client-Tool-Action Terminal** (emits `finish_reason: "tool_calls"`; participates in the client's multi-turn loop). Per ADR-033 (callee resolution of OQ #26) + ADR-034 (terminal + artifact-bridge). This is the north-star "delegate work, apply locally" surface; its long-horizon coherence (axis 2) is a recorded BUILD/PLAY validation target under ADR-097 Conditional Acceptance.

**Cycle 7 loop-back additions** (full module entries in [system-design.agents.md](./system-design.agents.md)):
- **L2 Loop Driver** — the layer-A control structure no ADR-027 component held; per-turn next-action decision + tool-mapping + swappable seat-filler LLM (Model Profile) + per-turn callee delegation + axis-2 `TurnDecision` diagnostic events per ADR-033.
- **L2 Single-Step Enforcer** — batch-truncation (D2-selected; the only candidate with τ′ evidence, model-independent so the seat-filler stays swappable); the structural grounding guarantee per ADR-033.
- **L2 Artifact Bridge** — reads `SessionArtifactStore.read_deliverable()` (new accessor) and marshals deliverable content into the client tool-call with fidelity; the F-ρ.1 marshalling step, distinct from the ADR-030 "Bridge mechanism", per ADR-034.
- **L3 Client-Tool-Action Terminal** — re-introduces `ClientToolCall` emission (the four pieces `0a7a822` removed) on the tool-driven path; multi-turn loop participation (consumes `role: "tool"`, routes to Loop Driver) per ADR-034.
- **Serving Layer extended** — surface-mode discrimination (D1; validate-not-assumed per advisory 2); composes the Client-Tool-Action Terminal; injects the seat-filler LLM into the Loop Driver.
- **Session Artifact Store extended** — `read_deliverable()`, the first read-side API on the formerly write-only store (advisory 4).
- **Eleven new fitness criteria FC-42 through FC-52**; **nine new dependency edges** (all cycle-free); **no invariant change** (AS-9 §Scope boundary annotation already records the conditional resolution of the loop-driver scope question).

**Cycle 7 loop-back #5 additions (ADR-037 — session termination; full entries in [system-design.agents.md](./system-design.agents.md)):**
- **L1 Session Action Record** — the framework-owned digest's home: per-session action records (action kind, target file path, client result) accumulated from the framework's own emissions and joined with client tool results — never reconstructed from client-serialized messages (Spike θ round 1 measured that reconstruction's failure). The record schema is the **extensible meta-record seam**: the write-log is the first increment; the false-stop share is the extend-on-evidence enrichment trigger (the practitioner's pre-mortem, given an address).
- **Loop Driver extended** — judgment-first trailing composition: every tool-result tail opens with a bare-form termination judgment (no tools, no client prompt) over the digest under the deliverable-accounting standard; COMPLETE returns the stripped judgment text as the protocol-clean finish turn (the model is the stop mechanism); REMAINING makes exactly one ADR-036-form action call with the judgment exchange discarded. `TurnDecision` gains `tail_kind` + `judgment_verdict` (the finish-policy fields — the event shape WP-LB-J was held for; **J unholds**).
- **Operator-Terminal Event Sink extended** — the TurnDecision INFO line carries the finish-policy fields; false-continue and false-stop shares are operator-computable from the serve log (the convergence-verified evidence ADR-037's Conditional Acceptance gating condition reads).
- **Orchestrator Configuration extended** — judgment-seat Model Profile resolution (defaults to the seat-filler's profile; shared profile = one re-validation run covers both instruments; hosted seat is an optimization slot, local path standing fallback).
- **Seven new fitness criteria FC-63..FC-69** (incl. FC-69 making the AS-3 turn-cap backstop refutable on the loop-driver surface — a pre-existing wiring gap the conformance scan surfaced); **one new dependency edge** (Loop Driver → Session Action Record, cycle-free); **no invariant change** (AS-9's §Termination-judgment instance annotation landed at DECIDE, Amendment #18).

**Cycle 7 additions** (full module entries in [system-design.agents.md](./system-design.agents.md)):
- **L2 Dispatch Pipeline** — orchestrates plan → dispatch → synthesize on the chat-completions surface; replaces the chat-completions handler's `OrchestratorRuntime` call path; owns the Plan → `InternalToolCall` adapter and pipeline-stage event emission per ADR-027. **Cycle 7 loop-back: scoped to the single-turn (non-tool) surface** by ADR-027's ADR-033 partial-update header.
- **L2 Routing Planner** — wraps the routing-planner system ensemble (`agentic-routing-planner`); produces typed `DispatchPlan` from chat-completions request content + capability list per ADR-028.
- **L2 Response Synthesizer** — wraps the response-synthesizer system ensemble (`agentic-response-synthesizer`); produces user-facing chat-completion response from structured `(REQUEST + PLAN + DISPATCH RESULTS)` input under strict-fidelity Rules 1-6 per ADR-029; streams synthesizer tokens via existing `OpenAiSseFormatter`.
- **L1 Capability List Builder** — produces the canonical capability list (capability ensembles only, filtered from validate-once-at-load registry); single source of truth consumed by Routing Planner and Capability Discovery Endpoint.
- **L3 Response Labeling** — three-layer honest served-by signaling (headers + body metadata; content-layer Rule 5 framing delegated to Response Synthesizer) per ADR-032 §Sub-promise (1); also emits `tool_choice_handling: "deferred"` bridge advisory per ADR-030.
- **L3 Capability Discovery Endpoint** — OpenAI-protocol-compatible capability discovery surface per ADR-032; BUILD picks one of `/v1/models` extension, sibling `/v1/ensembles`, or response-metadata-only.
- **Constitutional invariants AS-9 and AS-10** added (domain-model Amendment Log entries #13 and #14): AS-9 names role-shape structural-bounding property; AS-10 names request-shape commitment (chat-completions surface; no client-side opt-in mechanism).
- **Orchestrator Runtime preserved as architectural option** per ARCHITECT disposition (a) for conformance Finding 2 — no production caller post-Cycle 7 BUILD; ADR-001 + ADR-011 + ADR-022 system-prompt amendment retained as architectural commitments for future surfaces.

**Cycle 6 additions** (full module entries in [system-design.agents.md](./system-design.agents.md)):
- **L1 Dispatch Event Substrate** — owns `DispatchTiming` event + `dispatch_id` correlation; unified emission substrate per Inversion N+2; routes to two destinations.
- **L3 Operator-Terminal Event Sink** — line-oriented `key=value` log lines per event; tool-call-emit logging + inference-wait heartbeats (liveness signals); validate-once-at-load startup-warning surface.
- **L2 Orchestrator-Context Event Sink** — JSON-shaped structured observations prepended to next ReAct turn; end-of-session summary under `execution.json::dispatch_log`.
- **L1 Session Artifact Store** — owns Artifact-as-substrate; the `.llm-orc/agentic-sessions/<session_id>/<dispatch_id>/<deliverable>.<ext>` path layout; retention (`session` / `durable` / `ephemeral`); `output_substrate` / `output_retention` / `calibration_substrate_access` YAML field enforcement.

**Cycle 6 module extensions** — Orchestrator Configuration (ADR-022 system-prompt amendment + new observability/substrate config keys); Orchestrator Tool Dispatch (emits `DispatchTiming` events; constructs typed `DispatchEnvelope`; conditionally skips harness for substrate-routed dispatches per AS-7 amended); Result Summarizer Harness (skip honored via Tool Dispatch's interposition order); Calibration Gate (three evaluation surfaces under substrate-routing); Ensemble Engine L0 (new optional YAML fields; validate-once-at-load library cache).

**Shared type** — `DispatchEnvelope` (ADR-024) is the typed return value of `invoke_ensemble`; lives alongside `LlmOrcStructuralError` in shared types. Fields: `status`, `primary`, `structured?`, `diagnostics`, `errors?`, `artifacts?`.

**Layering rule.** Edges point from a higher layer to a same-or-lower layer; never upward — **with one narrow exception per ADR-016 (conditional acceptance; first-deployment evidence pending):** a read-only signal channel may flow from L0 (Ensemble Engine dispatch outputs) to L1 (Calibration Signal Channel module), restricted to calibration data and gated by five bounding mechanisms specified in ADR-016. The exception is signal-channel-specific (calibration only; not a general upward import permission) and read-only (no upward writes; ADR-002's layering rule for write paths is unchanged). All other layer pairs remain prohibited. FC-2 (static import check) and FC-3 (cycle detection) recognize the calibration-channel exception via an annotated allowed-edge in their layer map. Intra-L3 edges (Serving Layer ↔ Session Registry, Orchestrator Configuration; Cycle 6 adds Serving Layer → Operator-Terminal Event Sink) do not form cycles. Cycle 4's seven new dependency edges (Runtime → Conversation Compaction; Compaction → Ensemble Engine; Tool Dispatch → Tier-Escalation Router; Tier-Escalation Router → Calibration Gate; Tier-Escalation Router → Ensemble Engine; Calibration Gate → Calibration Signal Channel; Ensemble Engine → Calibration Signal Channel as the upward exception) are all verified cycle-free. **Cycle 6's 13 new edges** (the four producer→substrate edges; substrate→two-sink fan-outs; substrate-related Session Artifact Store + Tool Dispatch + Runtime edges; Serving Layer → Operator-Terminal sink; Operator-Terminal sink → Ensemble Engine for validate-once-at-load; conditional Calibration Gate → Session Artifact Store) are likewise verified cycle-free — the substrate's fan-out-to-sinks pattern is unidirectional and does not loop back to producers.

**Falsification trigger (ADR-016).** If BUILD or first-deployment evidence finds that mechanism (b) time-decay windowing or mechanism (d) periodic out-of-band audit dispatch cannot be operationalized within ADR-002's L0-L3 structure (e.g., the mechanisms require a top-level module orthogonal to the four-layer architecture), the elaboration-by-evidence framing commitment is invalidated, the reorganization branch re-opens, and ADR-016 is re-deliberated with reorganization on the table.

**Five-tool internal action surface (ADR-003).** The orchestrator's ReAct iterations call only: `invoke_ensemble`, `compose_ensemble`, `list_ensembles`, `query_knowledge`, `record_outcome`. No others. Client-declared tools flow through the response surface (turn-boundary delegation), not the action surface.

**Plexus is optional (ADR-002, AS-8).** L1's Plexus Adapter has no-op fallbacks exercised by stateless-mode tests (FC-7). The orchestrator works statelessly without Plexus; Plexus upgrades it to a learning system.

### L1 — Domain policy

| Module | One-line purpose |
|---|---|
| **Composition Validator** | Validates a proposed composed ensemble against the existing reference graph using the same routine as load-time validation (FC-6). Stricter than load-time: enforces "compose from existing primitives only" (AS-6). |
| **Budget Controller** | Enforces turn and token limits at each ReAct iteration boundary (FC-10). Control plane, not model plane (AS-3). |
| **Autonomy Policy** | Gates orchestrator actions against the Session's Autonomy Level (FC-11). Surfaces composition events when configuration requires it. |
| **Calibration Gate** *(extended in Cycle 4 per ADR-014)* | Tracks Calibration state and produces dispatch-time calibration verdict (Proceed / Reflect / Abstain trichotomy). ## Client Tool Surface Commitment

**Decision.** The orchestrator's **internal** tool surface is exactly the five ADR-003 tools and no others. Client-declared tools (the `tools[]` array on a `/v1/chat/completions` request) become the orchestrator's **response surface**:

- The orchestrator's ReAct iterations call only the five internal tools.
- When a task step requires a client-side action (bash, file edit, etc.), the orchestrator closes the current turn with `finish_reason: tool_calls` and emits one or more client-tool `tool_calls[]` in the completion response.
- The client executes the tools and sends `role: tool` messages back in the next `/v1/chat/completions`. The orchestrator resumes the same Session's ReAct loop with those messages as observations.

**Provenance.** Committed in ARCHITECT 2026-04-20 on user direction. Honors ADR-003 strictly ("no others" refers to *internal* action surface; delegation at the turn boundary is response-surface behavior, not internal action). Supersedes the `interaction-specs.md` open-boundary note for the Tool User stakeholder.

**Scenario gate resolved (2026-04-22; Amendment #4).** The four stress scenarios from roadmap Open Decision Point #1 are written into `scenarios.md` §Client Tool Surface Commitment. All four are carried by Option C:

- Scenarios (a) and (b) — turn-boundary delegation path, Session continuity across a client-tool round trip.
- Scenario (c) — first-agent client-filesystem-file need carried by *pre-invoke* delegation: orchestrator reads at the prior turn boundary, folds into `invoke_ensemble`'s `input_data`.
- Scenario (d) — composed ensemble's un-predicted mid-execution client-tool need carried by the **retry pattern**: ensemble runs to completion; agent emits structured `needs_client_tool` signal; Result Summarization preserves it; orchestrator observes, closes next turn with client-tool delegation, re-invokes with client-tool result folded into `input_data`. The DAG engine never suspends; Layer 3 is unchanged.

Option D (mid-execution callback) would require amending ADR-001/ADR-002 and adding suspend/resume to the synchronous DAG phase loop. Out of scope for Cycle 1; reopens only at the architectural ADR level. The retry pattern is conditional on composed ensembles following a `needs_client_tool` convention; the enforcement mechanism is a build-time layering decision (roadmap Open Decision Point #8).
